Problem

Chef published a blog post, and is now receiving many queries about it. On day ith, he receives Qi queries. But Chef can answer at most k queries in a single day.Chef always answers the maximum number of questions that he can on any given day (note however that this cannot be more than k). The remaining questions (if any) will be carried over to the next day.Fortunately, after n days, the queries have stopped. Chef would like to know the first day during which he has some free time, i.e. the first day when he answered less than k questions.

Input

  • First line will contain T, the number of testcases. Then the testcases follow.
  • The first line of each testcase contains two space separated integers n and k.
  • The second line of each testcase contains n space separated integers, namely Q1, Q2, ... Qn.

Output

For each testcase, output in a single line the first day during which chef answers less than kk questions.

Constraints

  • 1≤T≤1051≤T≤105
  • 1≤1≤ sum of n over all testcases ≤105≤105
  • 1≤k≤1081≤k≤108
  • 0≤Qi≤1080≤Qi≤108

Subtask

  • Subtask 1 - 20% points - Sum of Qi over all testcases and days ≤3.106≤3.106
  • Subtask 2 - 80% points - Original constraints

Approach

We start the solution by gathering all the required details from the question, like it is mentioned that chef receives Qi queries every day which indicates we have to make an array. Size of array is n as it is given that after nth day queries stop, chef can only answer k questions in one day, if any number of questions remain, they should be answered next day along with fresh queries that come on that day. So, now chef wants to know after which day he would be free for some time, that is, on which day he would answer less queries than he canso that he can get some time off.

We will start by taking inputs as directed in the question. One approach could be:

As I have stated above i.e. by making an array wherein, we can store queries for each day. Answer the maximum number of queries that we can and pass remaining to next day, that is, we subtract queries that can be answered and add remaining to next day’s queries.

Instead of using an array we could use two temporary variables, one that stores query for that day and second stores copy of it. We can subtract maximum queries that can be answered on that day from second variable and add the remaining ones. We use that first temporary variable to take the input for the next day. We use a counter variable to count number of days for which queries were more than maximum. And keep on subtracting maximum from the variable that is storing remaining number of queries.

If for any day value of variable is than maximum number of queries that can be answered then this day is the first free day and we print counter variable’s value. We are using greedy approach, as we check if current day is the free day and then we move to next day other than taking sum of all queries and then dividing to find the answer.

Algorithm

  1. Input t
  2. While t is greater than 0
  3.   Input n and k
  4.   Initialize temporary(temp) variable to 0
  5.   Initialize counter(count) variable to 0
  6.   For I =0 to n
  7.    Input Qi
  8.    temp = temp + Qi
  9.    If (temp is less than k)
  10.     Print count
  11.     Break the loop
  12.    Else
  13.     temp = temp – k
  14.   If (temp is greater than k)
  15.    Print (n+(temp+1)/k)
